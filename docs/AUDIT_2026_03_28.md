## AudioContext Suspension Bug

**Root cause of "big chungus" model scoring silence at 0.80.**

The Console's `AudioRecorder.tsx` countdown timer (3× `setTimeout(1000)`) breaks
the browser user-gesture chain before `new AudioContext()`. Browser creates the
context in **suspended state**. `ScriptProcessorNode.onaudioprocess` fires normally
(correct chunk counts, correct durations, valid WAV headers) but
`getChannelData(0)` returns all zeros.

**Fix**: `await ctx.resume()` after creating AudioContext.

**Defense in depth added**:
- Server-side: `recordings.py` RMS check (MIN_RMS_ENERGY = 10.0), rejects with 400
- SDK: `_extract_temporal_embeddings` skips files with RMS < 1e-6
- Client-side: `audioQuality.ts` warns on RMS < 0.01

**Key reference**: The shipped "viola" temporal_cnn scores silence at 0.15 with
zero silence negatives. Any model scoring significantly higher has a data problem.
