# ViolaWake Proven Training Recipe

The canonical training recipe for production-quality wake word models. This pipeline produced the Viola temporal CNN (d'=8.577, EER=0.8%, AUC=0.9993) and is the required default for all Console website training jobs.

## The Proven Model

- **File**: `experiments/models/j5_temporal/temporal_cnn.onnx` (102KB)
- **Architecture**: TemporalCNN(96, 9) — 25,409 params, <5ms inference
- **Training script**: `experiments/train_temporal_j5.py`
- **Evaluation results**: `experiments/j5_temporal_results.json`
- **Head-to-head eval**: `experiments/head_to_head_eval.py`
- **Training data**: `experiments/embedding_cache_temporal.npz` (65MB, 13,146 embeddings)

### Results (seed 42, the exported model)

## Training Data Composition

13,146 total samples (1,495 positive, 11,651 negative):

| Tag | Count | Source |
|-----|-------|--------|
| `pos_main` | 1,378 | User recordings + TTS positives (20 voices x 3 phrases) + augmented variants |
| `pos_diverse` | 117 | Additional TTS diversity samples |
| `neg_main` | 10,175 | Universal corpus (LibriSpeech, MUSAN speech/music/noise) + speech negatives |
| `neg_confusable` | 325 | Round 1: 30 confusable words x 10 voices (broad phonetic coverage) |
| `neg_confusable_v2` | 1,151 | Round 2: 16 confusable words x 10 voices (tight variants, hard negatives) |

### Two-Round Confusable Negative Mining

The key differentiator. Two separate rounds of phonetically-similar words:

1. **Round 1** (broad): 30 confusable words x 10 TTS voices = ~300 clips
   - Phonetically similar words generated by LLM (e.g., "viola" -> "violent", "violin", "viloa", "fiola")
   - Broad phonetic coverage to prevent obvious false positives

2. **Round 2** (tight): 16 confusable words x 10 TTS voices = ~160 clips
   - Tighter phonetic variants (e.g., "vi-oh-la", "vee-ola", "viola!" with emphasis)
   - Hard negatives that push the decision boundary

## Exact Training Parameters

```python
# Architecture
TemporalCNN(embedding_dim=96, seq_len=9)  # 25,409 params

# Loss
FocalLoss(gamma=2.0, alpha=0.75, label_smoothing=0.05)

# Optimizer
AdamW(lr=1e-3, weight_decay=1e-4)

# Scheduler
CosineAnnealingLR(T_max=80)  # epochs

# Weight averaging
EMA(decay=0.999)  # with auto-select between raw and EMA weights

# Training
epochs = 80        # max, early stopping usually triggers earlier
batch_size = 64
patience = 15      # early stopping patience
seed = 42          # reproducible

# Data splitting
Group-aware stratified split (val_fraction=0.2)
# All augmented variants from the same source file stay in the same split
# Prevents data leakage from augmented copies

# Gradient clipping
clip_grad_norm_(max_norm=1.0)
```

## 8-Phase Pipeline (CLI Production)

The full pipeline that generates training data from scratch:

1. **User positives**: Raw WAV recordings from the user
2. **TTS positives**: 20 voices x 3 phrases ("word", "hey word", "ok word") + noise + reverb variants
3. **Audio augmentation**: audiomentations pipeline (21 copies per original: pitch shift, time stretch, noise, reverb, speed, gain, etc.)
4. **Confusable negatives R1**: 30 phonetically-similar words x 10 TTS voices
5. **Confusable negatives R2**: 16 tight phonetic variants x 10 TTS voices
6. **Speech negatives**: 104 common phrases x 5 TTS voices
7. **Universal corpus**: LibriSpeech + MUSAN (speech/music/noise), capped at 2000 per source
8. **TemporalCNN training**: FocalLoss + AdamW + CosineAnnealingLR + EMA + group-aware split

## Console Parity

The Console backend (`console/backend/app/services/training_service.py`) MUST use identical parameters:

- `_train_temporal_cnn` (NOT `_train_mlp_on_oww`)
- Confusable R1: `n_confusables=30, voices_per_word=10`
- Confusable R2: `n_confusables=16, voices_per_word=10`
- Speech negatives: `n_voices=5`
- TTS positives when user has <100 samples
- Universal corpus if available
- Same training hyperparameters (epochs=80, patience=15, etc.)

See `console/backend/scripts/train_full_pipeline.py` for the standalone production script.
