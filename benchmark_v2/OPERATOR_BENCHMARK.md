# Custom Wake Word Benchmark: "Operator"

**Date:** 2026-03-28
**Purpose:** Prove that ViolaWake's training pipeline generalizes to arbitrary wake words, not just "Viola."

## Premise

A wake word SDK is only useful if it works for *your* word, not just the vendor's demo word. This benchmark trains a custom "operator" model from scratch using only the public ViolaWake training CLI and evaluates it against the same methodology used in Benchmark v2.

## Training Protocol

```bash
# Step 1: Generate TTS positives (117 files across 20 Edge TTS voices x 3 variants)
violawake-train --word "operator" --output models/operator_v2.onnx --epochs 50
```

| Parameter | Value |
|-----------|-------|
| Architecture | Temporal CNN (9-frame context, 96-dim OWW embeddings) |
| Model size | 100 KB (25,409 parameters) |
| Training samples | 6,917 positives (augmented), 8,185 negatives |
| Augmentation | 10x factor (gain, time stretch, pitch shift, noise, time shift) |
| Loss function | BCE with AdamW (lr=0.001) |
| Weight averaging | EMA (decay=0.999) |
| Best epoch | 2 of 17 (early stopped, patience=15) |
| Training time | **89 seconds** on i7-12700H (CPU only) |

## Evaluation Corpus

## Cross-Word Comparison

Each system evaluated on its own best wake word with adversarial negatives specific to that word.

| Model | Wake Word | EER | ROC AUC | Training Time | Architecture |
|-------|-----------|-----|---------|---------------|-------------|
| **ViolaWake** | **viola** | **5.49%** | **0.988** | ~48s | Temporal CNN |
| **ViolaWake** | **operator** | **7.2%** | **0.984** | **89s** | Temporal CNN |
| OWW | alexa | 8.24% | 0.956 | N/A (pre-trained) | MLP on OWW embeddings |

## Key Findings

### 1. The Training Pipeline Generalizes

"Operator" achieves EER 7.2% — within 1.7 percentage points of "Viola" (5.49%) and better than OWW's best published word "alexa" (8.24%). This is a different word, different phoneme structure, different confusable set. The pipeline works.

### 2. Temporal CNN is the Differentiator

The MLP architecture (operator_v1) is catastrophically broken: it scores everything above 0.89, giving EER ~49%. The Temporal CNN (operator_v2) with identical OWW embeddings achieves EER 7.2%. The 9-frame temporal context is what enables discrimination — scoring a sliding window of embeddings rather than a single frame.

### 3. 89 Seconds to a Custom Wake Word

From zero data to a 7.2% EER model in 89 seconds (CPU only). The training CLI handles TTS sample generation, confusable negative generation, augmentation (10x), and temporal model training end-to-end. No manual data collection required.

### 4. OWW Cannot Detect Custom Words Without Retraining

OWW's pre-trained "alexa" model produced 0 true positives on "operator" audio (expected — different word). OWW provides no public training CLI to create custom wake words from scratch. ViolaWake's `violawake-train` fills this gap.

## Reproducibility

All scripts and data needed to reproduce this benchmark are in the repository:

```bash
# Regenerate the operator model from scratch
violawake-train \
  --word "operator" \
  --output models/operator_reproduced.onnx \
  --epochs 50

# Evaluate
violawake-eval \
  --model models/operator_reproduced.onnx \
  --test-dir eval_operator \
  --report
```

Training data (TTS positives and confusable negatives) is auto-generated by the training CLI. Exact reproduction requires the same Edge TTS voices and random seed.

## Limitations

- All test audio is TTS-generated (synthetic), not real-speaker recordings
- Real-world performance depends on deployment environment, microphone, and background noise
- The 89-second training time is CPU-only on a modern laptop; slower hardware will take longer
- Cohen's d = 3.50 indicates the model would benefit from more positive samples (recommend 500+)
- Confusable negatives are phonetic permutations, not drawn from real-world ambient audio

## Conclusion

ViolaWake's training pipeline produces competitive wake word models for arbitrary words in under 2 minutes. The Temporal CNN architecture consistently outperforms MLP heads on OWW embeddings, achieving sub-8% EER on both "viola" and "operator" — two phonetically different wake words with distinct confusable sets.
